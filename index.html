Understanding Ritual Architecture

🔮

Infernet

This is Ritual's core oracle network. Infernet connects off-chain AI computations to on-chain smart contracts. Any EVM-compatible chain can use it - without trusting centralized APIs.

⛓️

Ritual Chain

A blockchain designed ground-up for AI. Traditional chains cannot run AI models - they're too computationally heavy. Ritual Chain is optimized from day one for AI workloads with heterogeneous compute support.

🎼

Symphony Consensus

Ritual's unique consensus mechanism. Uses dual proof sharding and distributed verification. Nodes can run on any hardware - CPU, GPU, or TEE, democratizing participation across the network.

🔐

ZK Proofs & TEEs

For computational integrity and privacy. Zero-knowledge proofs ensure AI outputs are correct without revealing sensitive data. Trusted Execution Environments add an extra security layer for confidential computing.

🛍️

Model Marketplace

Developers get access to various AI models - LLMs, predictive models, and custom algorithms. Models are treated as digital assets - transparent, tradable, and verifiable on-chain.

🔌

Prover Network

Ensures computational integrity using ZKPs and TEEs. Whenever an AI task executes, provers verify that computation was performed correctly - making cheating cryptographically impossible.

Your Ritual Learning Journey

Level 1: Understand the Problem

Today's Web3 + AI integration is centralized. When you use AI in a blockchain app, you trust external APIs (OpenAI, Google). You don't know what the model is computing, what data is being used, or if results are manipulated. This goes against the core principles of Web3 - trustlessness and verifiability.

Level 2: Ritual's Solution

Ritual creates a decentralized compute layer where AI models run off-chain but their outputs are cryptographically proven on-chain using zero-knowledge proofs. The trust gap is eliminated - you can verify without revealing data.

Level 3: Architecture Deep Dive

Infernet: Oracle network that connects off-chain AI to on-chain contracts.
Ritual Chain: Purpose-built blockchain for heterogeneous AI compute.
Provers: Verify computational integrity using ZK and TEE technology.

Level 4: Technical Concepts

ZK Proofs: Proof without revealing data. You can prove computation is correct without showing actual data or process.
TEEs: Trusted Execution Environments - hardware-level security for sensitive computations.
Heterogeneous Compute: Different hardware (CPU, GPU, TPU) work together seamlessly.

Level 5: Start Building!

Now you're ready to build on Ritual. Use the Infernet SDK, integrate AI models into smart contracts, and create decentralized AI-powered dApps. The future is yours to build!

Start Building with Ritual

AI integration with Ritual is very simple. Here's how you can add AI to your smart contracts: 

// Calling an AI model using Infernet SDK

import { Ritual } from '@ritual/sdk';

// Initialize Ritual client
const ritual = new Ritual({
  apiKey: process.env.RITUAL_API_KEY,
  network: 'mainnet'
});

// Query the AI model
const response = await ritual.useInference({
  model: 'LLAMA2-30B',
  prompt: 'Analyze this DeFi protocol risk',
  parameters: {
    temperature: 0.7,
    maxTokens: 500
  }
});

// Response is on-chain verified with ZK proof
console.log(response.output);
console.log(response.proof); // ZK proof of computation
                
// AI integration in Smart Contract (Solidity)

pragma solidity ^0.8.0;

import "@ritual/contracts/IInfernet.sol";

contract AIDataAnalyzer {
    IInfernet public infernet;
    
    function analyzeData(string memory data) public returns (bytes32) {
        // Request AI computation
        bytes32 requestId = infernet.requestCompute(
            "LLAMA2-30B",
            abi.encode(data),
            address(this)
        );
        
        return requestId;
    }
    
    // Callback function - Infernet calls this with result
    function infernetCallback(
        bytes32 requestId,
        bytes memory output,
        bytes memory proof
    ) external {
        // Verify ZK proof
        require(infernet.verifyProof(proof), "Invalid proof");
        
        // Process AI output on-chain
        // Your logic here...
    }
}
